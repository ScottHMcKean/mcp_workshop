Section 1

What is MCP, and why now?

The Model Context Protocol is a JSON-RPC contract between AI hosts and the tools, data, and workflows they reach for. It is not a model and not a runtime — it is the wire.

DATABRICKS
From N×M to N+M

Before MCP, every host re-implemented every integration. After MCP, hosts and servers speak one protocol — the same shape that let LSP unify editors and language servers.

Before

N×M glue code

Each host built its own connectors to GitHub, Slack, your warehouse, your vector store.

Cursor ─┐ Claude Desktop ├──→ build all integrations Genie Code ├──→ for every host, separately your agent ─┘
After

N + M servers

Each host implements MCP once. Each tool ships an MCP server once. They compose.

Genie Code ─┐ Playground ├──→ MCP → Genie / VS / UC your notebook─┘ you.com / your App
USB-C of LLM context.  Open spec · Anthropic shipped first · OpenAI, Google, Microsoft followed in 18 months.
DATABRICKS
What MCP actually is

JSON-RPC 2.0 over a transport. That is the entire wire format. Everything else is convention.

→ {"method": "tools/list"} ← {"tools": [{"name": "execute_sql", "inputSchema": {...}}, ...]} → {"method": "tools/call", "params": {"name": "execute_sql", "arguments": {"query": "SELECT 1"}}} ← {"content": [{"type": "text", "text": "[{\"_c0\": 1}]"}]}

What it is not: not a runtime (no code execution), not a model (provider-agnostic), not a framework (FastMCP is a framework — MCP is the wire), not "function calling" (function calling is a model API; MCP decouples tools from any one model).

Four roles

Host fans out. Servers don't talk to each other. Roots tell servers which workspace they are scoped to.

Server Exposes capabilities (tools, resources, prompts) Genie Spaces · Vector Search · UC Functions · bakehouse-detective

Three primitives

Most servers ship only tools. They miss half the spec. Resources and prompts are where the leverage is — and where Databricks Apps differentiate from managed servers.

Model picks

Tools

Model reads your description and decides when to call. JSON-Schema args, JSON return.

franchise_sales_trend(franchise, days) — daily revenue, anchored to dataset window.

App picks

Resources

Host shows them in a picker. User attaches to context. Model never reads 5,000 franchises into its window.

franchise://list — markdown index of all 48 franchises.

User picks

Prompts

Slash-command UX. Server expands a templated workflow. Opinions belong here, not in tool docstrings.

/diagnose_franchise austin — orchestrates four tools in order.

DATABRICKS
Beyond tools — the bidirectional bits

Three things that turn the host↔server conversation two-way. We demonstrate the first two in Section D's notebook, and use sampling live in Section E.

📝

Sampling

Server calls back to the host's LLM. No API key on the server. Host pays the bill.

ctx.session.create_message(...) inside compose_brief.

💬

Elicitation

Tool needs missing structured input? Don't fail or hallucinate — ask the user mid-call.

ctx.elicit(message=..., schema={enum: [low, med, high]})

🔔

Notifications

Resources can change. Server pushes, host re-fetches. No polling.

ctx.session.send_resource_updated(uri=...)

Transports & auth

Two transports (one deprecated). Auth is the transport's problem. On Databricks, the browser session is the credential — no tokens to paste.

TransportWhen to use it
stdio Local servers spawned by the host. Used by desktop hosts (Cursor, Claude Desktop, Claude Code).
Streamable HTTP Remote / hosted servers. Resumable sessions. Use for production. Databricks Apps speak this.
SSE Deprecated. Streamable HTTP replaces it.
Identity passthrough on Databricks Apps · x-forwarded-access-token + x-forwarded-email → every UC query runs as the user. RLS / CLS apply automatically.

The MCP landscape on Databricks

Hard cap to remember: a host session can attach at most 20 tools across all MCP servers. Don't connect every Genie Space and every UC schema.

Managed

Auto-available

Databricks operates these. Workspace identity authorizes you. Zero setup.

  • Genie Spaces — text-to-SQL on curated tables
  • Vector Search — every index you can read
  • UC Functions — every function you can read
External

Add by URL

Public servers from SaaS vendors. Bearer / OAuth. Every external server is a new trust surface.

  • you.com, Brave — web search
  • Slack, Notion, GitHub — first-party
  • Databricks Marketplace MCP category
Custom App

You deploy it

FastMCP behind a Databricks App. Sampling, elicitation, side effects, identity passthrough.

  • Capabilities the managed servers don't have
  • Workspace OAuth — no tokens to manage
  • Section E — you'll deploy one

Two in-workspace hosts

Same managed MCPs available in both. Different UX, different ergonomics. We use both.

Host

Genie Code

Released March 2026, replaced Databricks Assistant. Ambient chat in the notebook / SQL editor sidebar.

  • UX — "help me while I work"
  • Trace — hidden by default
  • Best for — finishing the task, not auditing the model
  • Sections A & E — first touch, last touch
Host

AI Playground

Explicit chat surface for tool + prompt experimentation. Pick the model, attach servers, watch the wire.

  • UX — "show me what the model decided to call"
  • Trace — tools/call args + results visible
  • Best for — iterating on prompts and tool descriptions
  • Section B — the trace is the value
Outside the workspace: Cursor, Claude Code, Claude Desktop, VS Code — the same managed servers reach those hosts via Streamable HTTP. Mentioned for context, not in the workshop's path.
DATABRICKS
Genie is a tool, not the agent

The mistake is making Genie itself the agent. Genie does one thing well — text-to-SQL on a curated space. The leverage is composing it with other tools inside one model turn.

your agent ├─ ask_genie: "Crumbly Creations daily revenue, full window" ├─ query_vs_index: "Austin reviews mentioning waiting / closed" ├─ you_search: "Austin road closures May 2024" ├─ compose_brief: # sampling — uses the host's LLM └─ log_finding: # side effect — writes to workspace.default.findings

Rule of thumb: if it's read or compute over Delta, make it a UC function. Reach for a custom FastMCP server only for sampling, elicitation, resources, or non-Delta side effects.
